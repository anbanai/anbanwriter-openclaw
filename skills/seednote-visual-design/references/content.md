# 种草笔记内容图设计规范

## 两类图片目标

| 图片类型 | 优化目标 | 内容策略 |
|----------|----------|----------|
| **封面** | 点击率 | 见 cover.md |
| **内容页** | 完读率 | 每页 2-3 个结构化信息点 |

尾图设计规范见 [tail.md](tail.md)。

---

## 内容页设计规范

**内容密度**：
- 文字占比：20-35%
- 每页字数：≤60 字
- 每页信息点：2-3 个，每点 ≤15 字
- 留白：35-45%

**6 种布局模式**（按内容类型选）：
1. **编号清单** — 步骤、要素列举
2. **对比双栏** — Before/After，优劣对比
3. **步骤流程** — 有先后顺序的操作
4. **标注图解** — 图片+手绘说明
5. **数据卡片** — 数字/数据展示
6. **Q&A 对话** — 问答式知识

**视觉规范**：
- 字体：圆角/手写，避免黑体/等线
- 高亮方式：色块或荧光线标记，勿加粗
- 背景：真实照片场景（每张不同场景）
- 批注：手绘风格，与封面保持一致
- 色调：与封面统一
- 中文主题只允许出现简体中文；禁止英文翻译、拼音、乱码、伪词和自动补充外文标签
- 当页文字必须来自 image-plan 的"必须出现文字"，不得自行创造新术语或改写关键数字

**视觉多样性**（3 张以上内容图时）：
- 3+ 种不同实景背景
- 构图角度变化（正视/俯视/局部特写）
- 焦点位置错落（左/中/右交替）
- 批注样式轮换（不完全重复同一种）

**禁止**：
- 3D 渲染/纯色大背景
- PPT 式对称布局
- 科学海报风格
- 无结构文字墙
- 连续 3 张相同视觉元素
- 手机框/设备框
- 与当页主题无关的茶类、季节或产品
- 误导性冲泡动作，例如把"10秒出汤"写成"焖泡10秒"

## 图片内容规划（image-plan.md）

基于选题方向、最终标题和目标受众，在生成图片前先独立规划每张图的具体内容，写入 `$DIR/image-plan.md`。

**图片总数**：N 张（奇数，3-7 张），其中封面 1 张 + 内容图 N-2 张 + 尾图 1 张。所有图片逐张调用 `generate_image` 生成。

**格式模板**：

```markdown
# 图片内容规划

## 总体策略

- 主题方向: xxx
- 目标受众: xxx
- 图片内容定位: xxx
- 计划图片数量: N 张

---

## cover 封面

- 钩子: （≤10字）
- 辅助信息: （≤15字）
- 必须出现文字: （1-2 行简体中文主文案）
- 视觉主体: （能直接看出主题的实物/场景）
- 禁止元素: （英文、伪词、错别字、无关品类、医疗功效承诺）
- 验收标准: （0.5 秒内能读懂主题）

---

## image_01 [内容] 主题：xxx

- 信息点1: （8-15字）
- 信息点2: （8-15字）
- 推荐布局: 编号清单 / 对比双栏 / 步骤流程 / 标注图解
- 必须出现文字: （2-4 条简体中文短句）
- 视觉主体: （当页知识对应的实物/过程/对比对象）
- 禁止元素: （英文、伪词、无关主题、误导性参数）
- 验收标准: （文字准确 + 主体准确 + 与 image-plan 一致）

---

## tail [尾部] 类型：{follow|comment|traffic}

匹配依据：{为什么选这个类型}
- 内容点: （见 tail.md 规范，根据类型填充）
```

## Prompt 模板

**内容页**：
```
{page_content}

页面类型：内容图（{page_type}），种草笔记信息流内容
风格延续：{style}（与封面保持一致）

平台规范：
- 3:4 竖版构图
- 真实照片背景（与其他内容图不同场景）
- 2-3 个结构化信息点，每点≤15字
- 圆角/手写字体，荧光色块高亮核心词
- 手绘批注风格
- 文字占比 20-35%，留白 35-45%
- 禁止：纯色背景、PPT布局、3D渲染、设备框
- 只出现"必须出现文字"中的简体中文，不出现任何英文、拼音、翻译或乱码
- 不改写关键数字、茶类名称和冲泡动作

内容主题：{topic}
全文概要：{full_outline}
必须出现文字：{required_text}
视觉主体：{visual_subject}
禁止元素：{forbidden_elements}
验收标准：{acceptance_criteria}
```

**生成方式（逐张调用 generate_image）**：

按 image-plan.md 逐张调用 `generate_image` MCP 工具：
- 每张使用对应 section 的信息点填充 `{page_content}` 和 `{page_type}`
- `output_path` 设为 `$DIR/image_01.png`、`$DIR/image_02.png` ... `$DIR/image_0{N-2}.png`
- `ref_image_path` 传入封面路径保持视觉一致性
尾图单独生成见 tail.md

> **关键规则**：每张内容图必须使用对应信息点构造独立 prompt，确保内容差异化。通过封面参考图保持视觉一致性。
