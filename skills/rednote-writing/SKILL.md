# 小红书内容写作知识库

## 1. 标题规范

**基本要求**：≤20 字，关键词前置，情绪词优先于纯信息词。

**常用句式速查**：
- 利益驱动：`带你学会___`、`让你掌握___`、`马上 get ___`
- 对号入座：`小个子/初次备孕/大学生___`（直接写出目标用户身份标签）
- 痛点代入：`为什么___？`、`如何做___？`、`___怎么办？`
- 数据具象：`7天搞定___`、`45款___测评`、`6款___必备`
- 权威背书：`TED 推荐`、`人民日报｜___`、`国家出品___`
- 危机警示：`千万别___，否则___`、`想避免___？立即停止___`

**标题评分公式**（内部生成 3 个候选，自动选 Top 1）：
```
title_score = 情绪强度(0-3) + 关键词密度(0-1) + 字数优化(0-1) + 句式加分(0-1)
情绪强度: 含强情绪词→3, 中情绪→2, 弱情绪→1, 无→0
字数优化: 12-18字→1, 否则→0
句式加分: 疑问/否定/数字句式→1, 否则→0
```

详细公式与案例：[title-psychology.md](references/title-psychology.md)
写作技巧：[title-techniques.md](references/title-techniques.md)

## 2. 正文结构（三段式）

1. **开头钩子**（1-2 句）：制造共鸣或抛出用户痛点问题
2. **核心内容**（2-3 段）：观点 → 证据 → 补充/反方视角
3. **互动收尾**（1 句）：引导评论的问句或明确 CTA（如"你们怎么看？评论告诉我"）

**段落格式**：
- 短段落，每段 2-4 行，避免大段文字堆砌
- 适当使用 emoji 增加趣味性（尤其针对年轻女性受众）
- 语气因受众而异：年轻女性→幽默温柔感性；职场内容→专业严谨

## 2.5 正文格式（关键 Gotcha）

**平台陷阱**：小红书不渲染 Markdown。`**粗体**` 在用户手机屏幕上就是 `**粗体**` 四个字符，`---` 就是三个横杠。这是平台硬限制，不是风格偏好。

发布到平台的正文必须是**纯文本 + emoji + 符号**。内部文档（topic-analysis.md、source-analysis.md、image-plan.md）不发布，可用 Markdown。

**格式设计三原则**：

1. **丰富的 emoji 是视觉锚点** — 用户刷信息流时，emoji 先于文字被注意到。把 emoji 放在段落开头或关键词旁，引导注意力
2. **空行是唯一可靠分隔** — 没有 `<hr>`，没有 `---`，换行两次就是分段，这是唯一有效的视觉分隔
3. **格式跟随内容调性** — 知识干货用序号 emoji（逻辑感强）；情感故事用氛围 emoji（松散排列）；测评对比用表格式 emoji 对比（直观）

**三种内容类型范例**：

知识类（咖啡馆选址攻略）：
```
选址是开店成败的 ✨第一关✨ 📍

❀ 三个核心指标 ❀

1️⃣ 人流量 — 早高峰过人数 > 500 是 ✨基准线✨
2️⃣ 租金比 — 月租不超过预期月营业额的 ✨15%✨
3️⃣ 竞争密度 — 500m 内同类型咖啡馆 ✨≤2 家✨

💡 选址前先蹲点，连续 3 天统计早中晚客流
```

情感/生活类（旅行记录）：
```
第一次一个人去京都

到的那天下着小雨，祇园的石板路全是水光 🌧️

没有行程表，没有攻略，就这样走着

路过一家小店，老板不会说中文
我不会说日语
但他给我倒了一杯热茶 🍵

有些瞬间，不需要语言
```

测评/对比类（手机摄影 App 测评）：
```
用了半年终于敢说哪个 ✨最好使✨ 📱

✧ 摄影App实测对比 ✧

Snapseed  ✅ 免费 / 曲线精准 / 学习成本高
VSCO      ✅ 滤镜漂亮 / 有社区 / 订阅才能用高级功能
Lightroom ✅ 专业全面 / 适合 raw 格式 / 稍重

日常随手拍 → ✨Snapseed✨
追求氛围感 → ✨VSCO✨
认真摄影 → ✨Lightroom✨

别三个都装，根据自己定位 ✨选一个用熟✨
```

### Markdown → Emoji 替换映射表

生成内容时直接使用 emoji + 纯文本，不要使用 Markdown 语法。以下映射表帮助将 Markdown 思维转换为小红书原生格式：

**基础映射**：

| Markdown 语法 | 替换为 | 说明 |
|---|---|---|
| `**粗体文字**` | emoji 包裹（见下方上下文表） | 不保留星号 |
| `---` 水平线 | 空行 | 用空行替代视觉分隔 |
| `- 列表项` | 1️⃣2️⃣3️⃣ 序号 emoji | 有序感替代无序列表 |
| `> 引用文字` | 💡 前缀 | 引用变为金句 |
| `# 标题` / `## 标题` | 符号包裹（见下方标题映射表） | 不保留 # 号 |
| `**强调词**` | ✨xx✨ / 🔥xx🔥 / ⚡️xx⚡️ 上下文选择 | 加粗含义用 emoji 包裹传达 |

**标题/分段 → 符号映射**：

| 标题类型 | 替换样式 | 示例 |
|---|---|---|
| 主分段标题 | 符号包裹 | ❀ 三个核心指标 ❀ |
| 次级分段标题 | 符号包裹 | ✧ 选址注意事项 ✧ |
| 干货类标题 | emoji + 符号包裹 | 🔥 ❀ 选址攻略 ❀ |
| 情感类标题 | 柔和符号包裹 | 🌿 ～旅行感悟～ |

可选装饰符号集：❀❁ ✧✦ ✿❀ ～～ 「」 〖〗 【】
根据内容调性选择——干货用几何符号（✧✦、〖〗），情感用花形符号（❀❁、～），中性用直角符号（「」、【】）。

**粗体用途 → Emoji 上下文选择**：

| 粗体用途 | 替换 emoji | 示例 |
|---|---|---|
| 核心观点/金句 | ✨ | ✨选址是开店成败的第一关✨ |
| 提示/建议 | 💡 | 💡选址前先蹲点3天💡 |
| 警告/注意 | ⚠️ | ⚠️避开这三个雷区⚠️ |
| 数据/事实 | 📊 | 📊500m 内竞品 ≤2 家📊 |
| 推荐/好物 | 🔥 | 🔥这款真的绝了🔥 |
| 步骤/方法 | 🎯 | 🎯三步搞定选址🎯 |

**包裹规则**：
- 短语关键词（≤8字）：用包裹 `✨重点内容✨`，突出具体词汇
- 长句/段落标记（>8字）：用前缀 `💡 选址前先蹲点3天`，避免两侧 emoji 过于密集
- 行内嵌关键词：用包裹，如"价格 ✨直接砍半✨ 到手才几十块"

**内容调性 → Emoji 集自动匹配**：

| 内容调性 | Emoji 集 | 使用场景 | 排列风格 |
|---|---|---|---|
| 干货类 | 📍📊🎯💡✅ | 教程、攻略、知识分享 | 结构化、有序 |
| 情感类 | 🌿✨🌙💕🍃 | 旅行、生活、感悟 | 松散、氛围感 |
| 测评类 | 📱⚡️🔥💰👍 | 产品对比、好物推荐 | 表格式、对比 |

**调性自动检测规则**：根据标题关键词匹配——含"攻略/教程/方法/指南"→干货类；含"记录/感受/旅行/日记"→情感类；含"测评/对比/推荐/好物"→测评类。无明确信号时默认干货类。

映射表是指导原则而非死板规则，当映射不适用时使用自然判断。

## 2.6 字数压缩（强制步骤）

**平台限制**：小红书正文限制 ≤1000 字（含 emoji 和空格）。超过无法发布。

**压缩流程**（仅在字数 >1000 时触发）：

### 步骤 1：字数检测

生成完整正文后，统计字数（含 emoji、空格、换行）。若 ≤1000 字，跳过压缩。

### 步骤 2：信息点标记

扫描正文，标记所有核心信息点（具体数据、可操作建议、独特观点）。这些不可删除。

### 步骤 3：三级压缩（按优先级执行）

**优先级 1 — 保留层（不可删）**：
- 核心信息点和数据
- emoji（视觉锚点，删除后影响可读性）
- 开头钩子和互动收尾（结构必需）

**优先级 2 — 精简层（可压缩）**：
- 冗余的修饰语和形容词（"真的非常特别的好" → "很好"）
- 重复表达同一意思的句子（合并为一）
- 过渡句（"那么接下来我们看看" → 直接进入内容）

**优先级 3 — 裁剪层（可删除）**：
- 装饰性铺垫（"说实话"、"不知道大家有没有这种感觉"）
- 多余的互动引导（保留一个即可）
- 可合并的短段落（两个说类似事情的段落 → 一个）

### 步骤 4：合并与压缩

按优先级 2→3 逐步精简，每轮后重新计数。目标：≤950 字（留 50 字安全余量）。

### 步骤 5：验证

压缩后检查：
- [ ] 字数 ≤1000
- [ ] 所有核心信息点保留
- [ ] emoji 保留完整
- [ ] 三段式结构完整（钩子→内容→CTA）
- [ ] 语气一致，无明显删减痕迹

## 3. 写作语气与沉浸感

- **对话式第一人称**：像跟朋友聊天，用"我"和"你/集美/姐妹"，拉近距离
- **沉浸式场景开头**：感官细节或微故事开场（"上周去咖啡馆，无意间发现..."），不要抽象陈述（禁止"本文将介绍..."）
- **平台原生表达**：自然使用小红书惯用语（"针不戳"、"绝了"、"已上车"），不堆砌，不刻意

**反面模式（禁止）**：
- 论文腔："本文旨在探讨..."、"综上所述..."
- 第三人称客观叙述："用户可以通过..."
- "本文将介绍"式开头
- 过度专业术语堆砌（除非垂直专业赛道）

## 4. 话题标签

- **数量**：5-8 个（少则曝光不足，多则稀释权重）
- **组合**：核心大话题（1-2 个）+ 垂直话题（2-3 个）+ 长尾精准话题（2-3 个）
- **格式**：`#话题名` 放在正文末尾
- **选取原则**：优先平台热门话题；结合当下季节/热点；直击目标用户痛点

## 6. 受众定位

- **卖点差异化**：先找出与竞品相比与众不同之处，在文案中放大突出
- **人群画像**：确定受众后才能确定文案书写方式（语气、词汇、emoji 用量）
- **语言风格适配**：
  - 年轻女性：幽默感 + emoji + 温柔感性语气 + 网络流行词（"集美"、"针不戳"）
  - 30+ 消费女性：生活化 + 有代入感的品质叙述
  - 职场人士：专业严谨 + 数据/案例支撑
- **账号定位一致性**：人设需贯穿每一篇笔记，避免风格差异过大

## 8. 风格一致性原则

- **标题/正文/封面三者风格统一**：风格差异会让用户质疑笔记专业度
- **账号调性一致**：单个账号内文案风格不宜差异过大，否则用户对账号真实性产生怀疑
- **萌宠类账号**：多用 emoji、可爱词汇；**职场教育类**：严肃严谨、专业用词
- 标题与封面文字的字体风格需基于账号内容定（不要萌宠用厚重字体、职场用可爱字体）

## 9. 爆款改写

输入一条爆款笔记 ID，输出高贴合主题的可发布新笔记。支持 3 种改写模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| `style-only`（默认）| 保留原主题与互动机制，封面只参考风格/色调/信息层级，不复用具体元素 | 大多数情况 |
| `medium` | 保留主题方向，适度借鉴结构，内容重新设计 | 需要兼顾原帖热度和原创度 |
| `tight` | 保留同主题、同互动机制、同内容结构，仅替换具体措辞和案例 | 测款阶段快速复制爆款结构 |

### 9.1 模板提取维度（5 维）

分析源笔记时提取以下 5 维信息：
- **标题模板**：年份/动作词/情绪词/句式
- **封面模板**：主文案、信息层级、配色、字体大小
- **正文模板**：开场金句、段落数量、结尾 CTA
- **互动模板**：评论区高频动作词、参与门槛
- **标签模板**：核心话题 + 长尾话题

### 9.2 改写规则

**tight 模式**（高一致性）：
- 保留：同主题、同互动机制（如"评论区打确认"）、同内容结构
- 替换：具体措辞、案例细节、账号人设口吻
- 禁止：逐句照抄、原图二次使用、原作者专属信息

**style-only 模式**（默认）：
- 封面：仅参考风格、色调、信息分层与竖版构图
- 明确禁止复用：人物姿势、图标组合、文本框形状与位置
- 使用"保留主题 + 重设计元素"策略

### 9.3 标题生成

内部生成 3 个候选，使用本 SKILL 第 1 节标题评分公式自动选 Top 1（≤20 字）。

### 9.4 内容输出格式

1. **标题**：自动评分选定的最优标题（≤20 字）
2. **正文**：1 版可直接发布，遵循本 SKILL 正文结构（不含话题标签）
3. **话题**：5-8 个（单独列出）
3. **字数限制**：≤1000 字。生成后若超字数，按第 2.6 节压缩流程处理

### 9.5 合规检查

加载第7节违禁词库，扫描标题与正文：

- **高风险词**：删除所在句子或重写段落，超过 3 处触发全文重写
- **中风险词**：自动替换为合规近义词
- **低风险词**：自动替换或删除

生成 `$DIR/compliance-report.md` 记录所有修改。
