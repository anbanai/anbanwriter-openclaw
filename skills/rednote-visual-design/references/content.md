# 小红书内容图设计规范

## 两类图片目标

| 图片类型 | 优化目标 | 内容策略 |
|----------|----------|----------|
| **封面** | 点击率 | 见 cover.md |
| **内容页** | 完读率 | 每页 2-3 个结构化信息点 |

尾图设计规范见 [tail.md](tail.md)。

---

## 内容页设计规范

**内容密度**：
- 文字占比：20-35%
- 每页字数：≤60 字
- 每页信息点：2-3 个，每点 ≤15 字
- 留白：35-45%

**6 种布局模式**（按内容类型选）：
1. **编号清单** — 步骤、要素列举
2. **对比双栏** — Before/After，优劣对比
3. **步骤流程** — 有先后顺序的操作
4. **标注图解** — 图片+手绘说明
5. **数据卡片** — 数字/数据展示
6. **Q&A 对话** — 问答式知识

**视觉规范**：
- 字体：圆角/手写，避免黑体/等线
- 高亮方式：色块或荧光线标记，勿加粗
- 背景：真实照片场景（每张不同场景）
- 批注：手绘风格，与封面保持一致
- 色调：与封面统一

**视觉多样性**（3 张以上内容图时）：
- 3+ 种不同实景背景
- 构图角度变化（正视/俯视/局部特写）
- 焦点位置错落（左/中/右交替）
- 批注样式轮换（不完全重复同一种）

**禁止**：
- 3D 渲染/纯色大背景
- PPT 式对称布局
- 科学海报风格
- 无结构文字墙
- 连续 3 张相同视觉元素
- 手机框/设备框

## 图片内容规划（image-plan.md）

基于选题方向、最终标题和目标受众，在生成图片前先独立规划每张图的具体内容，写入 `$DIR/image-plan.md`。

**图片总数**：N 张（奇数，3-7 张），其中封面 1 张 + 内容图 N-2 张 + 尾图 1 张。封面和尾图单独生成，内容图使用 `generate_images` 批量生成（count = N-2）。

**格式模板**：

```markdown
# 图片内容规划

## 总体策略

- 主题方向: xxx
- 目标受众: xxx
- 图片内容定位: xxx
- 计划图片数量: N 张

---

## cover 封面

- 钩子: （≤10字）
- 辅助信息: （≤15字）

---

## image_01 [内容] 主题：xxx

- 信息点1: （8-15字）
- 信息点2: （8-15字）
- 推荐布局: 编号清单 / 对比双栏 / 步骤流程 / 标注图解

---

## tail [尾部] 类型：{follow|comment|traffic}

匹配依据：{为什么选这个类型}
- 内容点: （见 tail.md 规范，根据类型填充）
```

## Prompt 模板

**内容页**：
```
{page_content}

页面类型：内容图（{page_type}），小红书信息流内容
风格延续：{style}（与封面保持一致）

平台规范：
- 3:4 竖版构图
- 真实照片背景（与其他内容图不同场景）
- 2-3 个结构化信息点，每点≤15字
- 圆角/手写字体，荧光色块高亮核心词
- 手绘批注风格
- 文字占比 20-35%，留白 35-45%
- 禁止：纯色背景、PPT布局、3D渲染、设备框

内容主题：{topic}
全文概要：{full_outline}
```

**生成方式（批量，必须使用 generate_images）**：

调用 `generate_images` MCP 工具，count 设为 N-2，传入封面作为参考图（ref）。
输出自动命名为 image_01.png, image_02.png ... image_0{N-2}.png
尾图单独生成见 tail.md

> **关键规则**：内容图必须用 `generate_images` 批量生成，严禁逐张调用，否则视觉一致性无法保证。
